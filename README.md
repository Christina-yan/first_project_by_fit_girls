## Описание проекта
### Краткое описание
Проект представляет собой модульное десктопное приложение на языке Python с использованием библиотеки Pygame, которое является сборником мини-игр, выполненных в Рождественской стилистике. 
Приложение скомпилировано в исполняемый файл (.exe) для ОС. 
### Иерархия файловой системы
Проект использует вложенную структуру для разделения кода запуска и контента.

Модуль управления (main.py). 
Является родительским процессом для всего приложения.

Функционал:
1) Инициализация окна приложения и глобальных настроек Pygame.
2) Отрисовка интерфейса главного меню. 
3) Импорт модулей из пакета games.

Маршрутизация: при выборе игры вызывается главная функция соответствующего файла.

Пакет мини-игр (games/): Каждая игра представляет собой автономный модуль с собственным игровым циклом.

А. Логика переходов (управление состоянием)

Вход: Запуск игровой функции инициализирует локальные переменные (счётчик, положение объектов).

Выход (ESC): Во всех играх реализован прослушиватель событий клавиатуры (pygame.KEYDOWN). При нажатии клавиши Esc: Игровой цикл прерывается (break). 
Управление возвращается в main.py.

Б. Особенности реализации игр
### 2048:
    Матричная логика перемещения чисел.
    Сохранение: Рекорд (лучший результат) сохраняется в локальном файле и загружается при запуске модуля для отображения в интерфейсе самой игры.
### Water Sort:
    Логика стеков (LIFO — Last In, First Out) для проверки переливания цветов.
    Сохранение: Сохраняется номер пройденного уровня.
### Santa’s Ride (Flappy Bird):
    Динамическая обработка физики и столкновений в реальном времени.
    Сохранение: Счёт сохраняется и отображается на экране при проигрыше.
### Pairs (Memory):
    Интерактивная сетка карточек. Таймеры для отображения несовпадающих пар.
    Особенность: Игра сохраняет рекорды, но каждая сессия начинается с нуля.

### Работа с данными и ресурсами
Загрузка изображений и звуков осуществляется с использованием относительных путей.
Нюанс компиляции: при сборке в .exe используется системный модуль os и sys, чтобы программа корректно находила изображения как в режиме разработки, так и внутри скомпилированного файла.
Для игр «2048», «Water Sort» и «Santa’s Ride» реализована система ввода-вывода (I/O).
При запуске игрового модуля происходит чтение файла сохранения.
Если текущий счёт игрока превышает сохранённый, файл перезаписывается новым значением.
Данные рекордов инкапсулированы внутри игр (не передаются в main.py).
### Технический стек и сборка
    Библиотека: Pygame.
    Интерпретатор: Python.
    Компоновщик: PyInstaller

Для того, чтобы запустить игру, есть 2 варианта:
1) Можно скачать архив из GitHub и запустить файл main. 
Для этого понадобится любая IDE, поддерживающая python и библиотеку pygame. 
2) Установка exe файла также осуществляется в GitHub. 
Просто перейдите в раздел Code/Releases/3 tags/ver_2.0.0  и установите там exe-файл.

Инструкция по сборке проекта (.exe)

1)	Убедитесь, что вы находитесь в корневой папке проекта (там, где лежит main.py) и у вас установлен PyInstaller.
Если не установлен, выполните команду в терминале “pip install pyinstaller”.
2)	Команда сборки
Мы упакуем код, папку games (картинки, звуки, скрипты игр) и папку sounds-for-menu. 
Скопируйте и выполните эту команду в терминале: 
pyinstaller --noconsole --onefile --icon="icon.png" --name="Christmas.minigames" --add-data "games;games" --add-data "sounds-for-menu;sounds-for-menu" main.py
3)	Где искать результат
После завершения процесса в папке проекта появится папка dist, внутри неё будет лежать готовый файл Christmas.minigames.exe.
4)	Как это будет работать у пользователя
Когда вы запустите этот .exe файл игра распакует свои внутренние ресурсы (картинки, звуки) во временную память. При попытке сохранить или загрузить рекорд, игра создаст новые .txt файлы рядом с .exe файлом. Таким образом, прогресс игрока будет сохраняться на его компьютере
